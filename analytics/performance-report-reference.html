Les données d'analyse sont envoyées automatiquement par les lecteurs Brightcove, y compris celles fournies par les SDK Native Player. Si vous n'utilisez pas Brightcove Player pour diffuser des vidéos Video Cloud, vous devez instrumenter le lecteur que vous utilisez pour envoyer les données au collecteur de données. Voir la présentation: API de collecte de données sujet pour plus de détails.

Pour accéder au rapport des performances, ouvrez le module de statistiques et sélectionnez Performance dans le volet de gauche. Le rapport présente un graphique qui affiche le nombre total de visionnages par jour pour la période choisie. Vous pouvez choisir une autre mesure qui vous intéresse dans la liste déroulante :

  • Vues vidéo - Nombre de fois où une vidéo a commencé à jouer, enregistrée au démarrage du flux (n'inclut pas le rembobinage ou la relecture) ; il ne s'agit pas d'une mesure des spectateurs individuels ; pour les vidéos comportant des publicités pré-roll, une vue n'est pas enregistrée tant que la vidéo commence à prendre. jouer
  • Video Impressions: le nombre de fois où une vidéo a été chargée et prête pour interaction.
  • Play Rate: le pourcentage de vidéos qui ont été chargées et vraiment lues (le nombre de visionnages divisé par le nombre d'affichages).
  • Viewed Minutes: le nombre de minutes de visionnage de la vidéo.
  • Engagement Score: le pourcentage moyen de la vidéo vue par visionnage (calculé en additionnant tous les pourcentages regardés et en divisant le résultat par le nombre de visionnages de la vidéo).
  • Daily Unique Viewers: calculé en fonction de l'adresse IP et de l'utilisateur-agent des visiteurs. Brightcove a pris la décision de ne pas utiliser de cookie tiers car les cookies sont de plus en plus bloqués par les navigateurs. Daily Unique Viewers est disponible au niveau du compte ou de la vidéo ; cette métrique n'est pas disponible lors du filtrage par une vidéo (ou autre chose). Brightcove permet également de déterminer les utilisateurs uniques par rapport à une valeur de votre choix au lieu de leur adresse IP. Pour plus d'informations, consultez le document développeur Suivi des événements statistiques.
  • Bande passante vidéo - Inclut la bande passante pour les vidéos uniquement. Notez que les données de bande passante sont retardées de 3 à 5 jours.
  • Bande passante totale: inclut la bande passante pour les vidéos, les images et les charges de lecteur. Notez que les données de bande passante sont retardées de 3 à 5 jours.

Par défaut, le rapport affiche les données de performances pour tous les types de flux. Si votre compte est activé pour Brightcove Live, vous pouvez choisir d'afficher les données de performance pour toutes les vidéos, les actifs VOD ou les actifs en direct. Clique le Type de flux pilule pour faire une sélection.

Notes :
  • Les rapports de statistiques ne comprennent pas les statistiques des vidéos supprimées de votre compte. Vous ne pouvez pas rechercher des vidéos supprimées dans le module de statistiques pour consulter un rapport sur une vidéo supprimée particulière. Pour obtenir un tel rapport, vous devrez connaître l'ID vidéo de la vidéo supprimée et utiliser le API Analytics pour générer le rapport.
  • Il faut jusqu'à 3 jours pour que les données sur les visionneuses uniques quotidiennes et les données de bande passante vidéo soient disponibles.
  • Une vue vidéo n'est pas la même que les vues à x% dans le rapport d'engagement. Un visionnage vidéo est compté lors de la première lecture de la vidéo. Les visionnages à 1 % sont enregistrés lorsque le premier % est regardé. La différence ici est que si le premier pourcentage est regardé plusieurs fois (si la vidéo est réglée sur la relecture automatique, par exemple), le nombre augmente, alors qu'une vue vidéo est comptée lorsque la vidéo commence à lire mais n'est pas comptée sur une relecture (sauf si l'utilisateur recharge la page ou regarde une autre vidéo dans la liste de lecture, etc.).
  • Les demandes de rapport simultanées dans le même compte s'exécuteront en série. Si les rapports ont de grandes plages de dates et que de nombreux champs sont sélectionnés, il peut y avoir un délai avant que tous les résultats ne soient renvoyés. Une bonne alternative consiste à planifier des rapports volumineux.
  • Avec les expériences In-Page, même si la ou les vidéos semblent être chargées dans un lecteur, les vidéos ne sont pas chargées tant que le spectateur n'a pas cliqué sur la superposition du bouton de lecture. Par conséquent, les impressions vidéo ne seront pas enregistrées par le module Analytics tant qu'un spectateur n'a pas lu une vidéo.

Le commutateur de rapports situé sous le graphique peut être utilisé pour passer rapidement à un autre rapport.

La liste déroulante à droite du commutateur de rapports peut être utilisée pour sélectionner une dimension de rapport. Les dimensions suivantes sont disponibles avec le rapport sur les performances :

  • Vidéo
  • lectuer
  • Source de trafic
  • Afficher le domaine
  • Pays
  • Appareil
  • Type d'appareil
  • Système d'exploitation de l'appareil
  • Fabricant de l'appareil
  • Type de navigateur
  • Jour

Notez que les dimensions disponibles changeront en fonction du rapport sélectionné. Au bas du rapport, vous trouverez des contrôles qui vous permettent de sélectionner le nombre de lignes à afficher et de naviguer vers d'autres pages.

Compare to previous period

Cocher la case Compare to previous period ajoute un graphique à lignes segmentées superposé qui affiche les données de la période précédente.

La période précédente est fonction de la période de dates sélectionnée dans le rapport. Par exemple, si vous sélectionnez une plage de dates des 7 derniers jours, la période précédente aura lieu il y a deux semaines. Il est conseillé de sélectionner des plages de dates qui correspondent à des semaines entières si vous désirez comparer des tendances sur plusieurs semaines. Par exemple, si vous sélectionnez une plage qui commence un lundi, la date de fin devrait être un dimanche. Si la plage commence un vendredi, faites-la conclure un jeudi. Vous pouvez également survoler les barres et la ligne segmentée avec votre curseur pour afficher la date et les valeurs correspondantes.

Détails du rapport de performance

La première colonne qui s'affiche dans le corps du rapport varie selon l'échelle sélectionnée (nom de la vidéo, nom du lecteur, pays, type de dispositif, etc.). Tous les rapports affichent les données suivantes :

  • Video Views: le nombre de fois où la lecture de la vidéo a été lancée. Cette statistique est recueillie au lancement du flux et ne comprend pas les retours en arrière ou les relectures. Il ne s'agit pas d'une mesure par visiteur individuel.
  • Video Impressions: le nombre de fois où une vidéo a été chargée dans un lecteur et a été prête pour interaction.
  • Play Rate: le pourcentage d'utilisateurs qui ont lu la vidéo une fois celle-ci chargée (il s'agit du nombre de visionnages de la vidéo divisé par les affichages vidéo).
  • Viewed Minutes: le nombre total de minutes de visionnage de la vidéo.
  • Engagement Score: le pourcentage moyen de la vidéo vue par visionnage (calculé en additionnant tous les pourcentages regardés et en divisant le résultat par le nombre de visionnages de la vidéo).

Les Vidéo et Jour les dimensions afficheront également le Téléspectateurs uniques quotidiens qui correspond au nombre moyen de téléspectateurs uniques quotidiens pour la période sélectionnée. Ceci est calculé à l'aide de l'adresse IP et de l'agent utilisateur des téléspectateurs.

Traffic Sources

Lorsque vous consultez les performances par sources du trafic, les sources possibles sont les suivantes :

  • Referral: l'utilisateur a cliqué sur un lien vers la vidéo.
  • Direct: l'utilisateur a saisi le lien à la main ou le référent n'a pas pu être identifié.
  • Secure search: l'utilisateur s'est connecté à un compte Google, a lancé une recherche et a cliqué sur un résultat (standard ou parrainé ; les informations de recherche ne sont pas précisées dans ce cas).
  • Organic search: l'utilisateur a lancé une recherche et a cliqué sur un résultat pour afficher la vidéo.
  • Paid search: l'utilisateur a lancé une recherche et a cliqué sur un résultat parrainé pour afficher la vidéo.

Affichage des analyses vidéo

Lorsque l'échelle Video est sélectionnée, un graphique affiche le nombre de visionnages par jour sur la période choisie. Le corps du rapport présente le nom de la vidéo, l'ID de vidéo, la durée et les détails des performances pour chaque vidéo.

Cliquez sur le lien d'une vidéo pour obtenir les performances spécifiques de cette vidéo.

Note: Si une vidéo qui ne se trouve pas dans votre bibliothèque Video Cloud est lue dans un lecteur Video Cloud (par exemple, une ressource distante ou la vidéo réservée que vous voyez dans le lecteur d'aperçu dans le module Lecteurs), aucun identifiant vidéo associé ne sera affiché.

Affichage des analyses des lectuers

Lorsque l'échelle Player est sélectionnée, un graphique affiche le nombre de visionnages par jour sur la période choisie. Le corps du rapport présente les données de performances pour chaque lecteur.

Cliquez sur le lien d'un lecteur pour obtenir ses performances spécifiques.

Note: Avant février 2016, les lecteurs utilisés dans les applications créées à l'aide du Brightcove Player SDK pour iOS ou Android s'affichaient avec un nom de lecteur de Lecture personnalisée ou native. À partir de février 2016, ces lecteurs sont désormais identifiés grâce aux valeurs de source et de destination précisées dans l'application. Pour plus d'informations, consultez soit le iOS ou Android Documentation du SDK.

Affichage de la source d'analyse du trafic

Lorsque l'échelle Source of Traffic est sélectionnée, un graphique affiche le nombre de visionnages par jour sur la période choisie. Le corps du rapport affichera les données de performance pour chaque source de trafic. Seules les catégories ayant des données pour la période sélectionnée seront affichées.

Cliquez sur le lien d'une des sources pour afficher les performances de celle-ci.

Affichage des analyses du domaine d'affichage

Lorsque l'échelle Display Domain est sélectionnée, un graphique affiche le nombre de visionnages par jour sur la période choisie. Le corps du rapport affichera les statistiques des domaines d'affichage. Le domaine d'affichage est le domaine sur lequel se trouvait le visiteur lorsqu'il a regardé la vidéo.

En cliquant sur le lien de n'importe quel domaine d'affichage, vous verrez les URL d'affichage de ces domaines.

Note: Si le lecteur se trouve dans un iframe, le domaine d'affichage sera enregistré comme URL de la page dans l'iframe.

Affichage des analyses de pays

Lorsque l'échelle Country est sélectionnée, un graphique affiche le nombre de visionnages par jour sur la période choisie. Le corps du rapport affichera les statistiques pour chaque pays où la vidéo a été vue. Seuls les pays ayant des données pour la période sélectionnée seront affichés.

Note: Brightcove utilise un service tiers pour nous fournir l'adresse IP et les informations de localisation. S'il n'y a pas d'entrée pour l'adresse IP en question, nous ne pourrons pas obtenir les informations sur le pays et Unknown apparaîtra.

Cliquez sur le lien correspondant à l'un des pays pour afficher les performances par région ou ville pour le pays sélectionné, si vous visionnez des analyses pour le compte, et non une vidéo individuelle.

Affichage des analyses de l'appareil

Lorsque l'échelle Device est sélectionnée, un graphique affiche le nombre de visionnages par jour sur la période choisie. Le corps du rapport affichera les données de performance pour chaque Type d'appareil - Système d'exploitation de l'appareil combinaison qui a été utilisée pour visionner une vidéo. Seuls les dispositifs ayant des données pour la période sélectionnée seront affichés.

Note: Le module Analytics détermine l'appareil à l'aide de l'en-tête de l'agent utilisateur. Si quelqu'un utilise un dispositif avec des headers usurpés, les données seront collectées selon le header déclaré. Le module Analytics ne fait aucun effort pour empêcher l'agent utilisateur d'être altéré.

Utiliser des filtres

Les filtres peuvent être utilisés pour limiter les données qui apparaissent dans le rapport. Pour ajouter un filtre, procédez comme suit.

  1. Clique le + Ajouter des filtres pilule en haut du rapport.
  2. Les données peuvent être filtrées par Mots clés , Les champs personnalisés , des pays et types d'appareils. Configurez les critères de filtrage à l'aide du panneau de filtrage. L'exemple suivant permet d'appliquer le filtre :
    • Balise = formation à la demande ET
    • Champ personnalisé Produit = Nuage vidéo ET
    • Pays = États Unis
  3. Cliquez sur Appliquer des filtres.

Les filtres appliqués apparaîtront sous forme de pilules en haut du rapport. Chaque pilule affichera un type de filtre et jusqu'à deux valeurs. Jusqu'à trois pilules seront affichées.

Les pilules supplémentaires sont regroupées en une pilule de débordement et sont affichées dans une info-bulle au survol.

Pour supprimer un filtre, cliquez sur une pilule et effacez le filtre du panneau des filtres.

Remarques sur l'utilisation des filtres

  •  Le type de flux n'est disponible que si Brightcove Live est activé sur le compte
  • La section des champs personnalisés sera désactivée si le compte n'est pas activé pour les champs personnalisés
  • Les filtres peuvent être ajoutés / supprimés en utilisant les boutons plus / moins à droite du type de filtre
  • Bien que les filtres semblent être appliqués, ils ne s'appliquent pas à toutes les sections du module Analytics
    • Les balises et les filtres de champ personnalisés ne sont pas appliqués lorsque vous avez exploré une vidéo ou un lecteur spécifique (la seule exception étant les dimensions Lecteur par vidéo ou Jour)
    • Graphique d'engagement dans le rapport d'engagement
  • Le chargement des données Analytics peut prendre un peu plus de temps lors de l'application des filtres
  • Type de filtre
    • Tous - Doit inclure tous valeurs
    • Tout - Peut inclure tout valeurs
    • Rien - Doit inclure rien des valeurs choisies

Exportation des données de rapports

Les données de rapports peuvent être téléchargées en tant que valeurs séparées par des virgules et des formats Excel. Notez que la fonction d'exportation enregistre les données directement à partir d'un API d'analyse appel. Les données exportées ne sont pas modifiées comme les données de l'interface utilisateur. Elles contiennent actuellement toutes les données inhérentes au rapport et pas seulement le sous-ensemble de champs de l'interface utilisateur. Voici quelques-unes des différences entre ce qui s'affiche dans l'interface utilisateur et dans les données exportées :

  • Des unités différentes (minutes contre secondes)
  • Des nombres arrondis
  • L'exportation de données contient les mesures d'engagement et de performances, peu importe où vous êtes dans l'interface utilisateur.
  • L'entrée Other/Third Party sera exportée sans ID de vidéo ou nom de vidéo associés.

Obtenir toutes les données au format de base depuis l'API vous permet de manipuler les données via d'autres méthodes que celles de l'interface utilisateur.

Les pages des rapports sont dans un format d'impression. Imprimer un rapport ou l'enregistrer en PDF donnera donc des pages avec une disposition parfaite.
