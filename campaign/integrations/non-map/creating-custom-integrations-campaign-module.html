Le module Campaign permet à Video Cloud de synchroniser les données de visualisation à partir de Campaign Brightcove Players aux plates-formes d'automatisation du marketing populaires. Le module CampaignCampaign peut également être configuré avec une intégration personnalisée. Avec une intégration personnalisée, les données de visionnage sont enregistrées localement dans Video Cloud et l'API Audience est utilisée pour extraire les données de manière programmée.

L'intégration du module Campaign se compose de deux composants principaux :

  • Un plug-in Brightcove Player pour permettre le suivi des données de vues vidéo
  • Développement personnalisé utilisant l'API Audience pour récupérer les données de visualisation

Exigences

Pour utiliser le module Campagne vous devez :

  • Avoir un compte Professional ou Enterprise Video Cloud
  • Activez le module Campagne pour votre compte en soumettant les informations requises dans l'écran de démarrage initial qui apparaît lorsque vous ouvrez le module Campagne
  • Cliquez sur le contrat de service la première fois que vous utilisez le module
  • Afin de recueillir les clients potentiels, vous devez configurer un formulaire de clients potentiels et l'attribuer à un lecteur.
  • Utilisez le code intégré avancé (dans la page) lors de la publication de votre lecteur ; le code d'intégration standard (iframe) ne fonctionnera pas car le plugin sera exécuté à l'intérieur d'un iframe et ne pourra donc pas accéder aux paramètres d'URL de la page parent

Configuration de la connexion

Suivez les étapes ci-dessous pour configurer une intégration personnalisée.

  1. Ouvrez le module Campagne et cliquez sur Connexions de données dans la navigation de gauche.
  2. Cliquez sur Add New Collection.
  3. Sélectionnez Custom pour l'intégration.
    Note: Les intégrations disponibles sont basées sur votre abonnement Video Cloud.
  4. (Facultatif) Entrez un nom de connexion.
  5. Cliquez sur Connect.

La connexion personnalisée est à présent configurée et prête à utiliser.

Configuration de l'intégration

Une fois la connexion établie, vous pouvez configurer l'intégration.

  1. Par défaut, l' état de la connexion sera activé. L' état de la connexion détermine si les données d'engagement vidéo Video Cloud sont enregistrées.
    • Activé - Les données sont collectées et seront enregistrées
    • Désactivé - Aucune donnée n'est collectée (l'intégration est désactivée)
  2. Sélectionnez une politique de conservation des données. La politique de conservation des données détermine la durée pendant laquelle les données de visualisation et les données du formulaire de prospect (si vous utilisez un formulaire de prospect Campaign) seront conservées dans votre compte Video Cloud.
    Note: Si vous sélectionnez Aucune rétention, AUCUNE donnée ne sera enregistrée. Lorsque vous utilisez l'API pour télécharger des données, il n'y en aura pas.

Création d'un lecteur compatible avec la campagne

Les données d'engagement vidéo ne seront enregistrées que lorsque vous utilisez un lecteur vidéo compatible avec Campaign. Pour plus d'informations sur la création d'un Brightcove Player activé pour la campagne, consultez Création d'un lecteur compatible avec la campagne.

Transmission d'un ID de suivi

Le module Campaign prend en charge la possibilité de transmettre un identifiant de suivi. L'ID de suivi peut être utilisé pour suivre un ID de campagne par exemple. L'ID de suivi peut être ajouté à l'URL qui affiche le lecteur activé pour la campagne ou ajouté au code d'intégration du lecteur.

  • Exemple: Ajoute à l'URL
  http://www.myurl.com/video.html ?bctrackingid=12345
  • Exemple: Ajouté au code d'intégration du lectuer
<vidéo data-video-id="6033402539001" data-account="1486906377" data-player="default" data-embed="default" data-application-id class="video-js" data-bc-trackingid="12345" les contrôles>< /vidéo>< script src="//players.brightcove.net/1486906377/default_default/index.min.js">< /scénario>

Identification des visiteurs

Lorsque vous utilisez une intégration personnalisée, vous êtes responsable de l'identification des visiteurs. Le plugin Campaign a besoin de deux informations clés :

Description Paramètre d'URL Paramètre d'intégration Valeur
Le visualiseur est-il connu (nécessaire pour le formulaire de prospect) bcknownuser data-bc-known-user 0 ou 1
Quel est le GUID (nécessaire pour le suivi des événements de vue) bccustomguid data-bc-custom-guid string
 

Ces valeurs peuvent être transférées via les paramètres d'URL ou le code d'intégration du lecteur. Pour que le suivi des visiteurs s'effectue, l'ID utilisateur doit être transféré en tant que paramètre d'URL sur la page qui affiche la vidéo. Le paramètre utilisateur connu est requis pour les formulaires de prospect (si vous souhaitez que le formulaire de prospect s'affiche conditionnellement pour les visionneuses inconnues).

Note: Pour que le lecteur/plugin lise les paramètres de l'URL, le code d'intégration Advanced (in-page) doit être utilisé. Le code intégré Standard (iframe) ne fonctionnera pas car le plugin s'exécutera à l'intérieur d'un iframe et ne peut donc pas accéder à l'URL de la page parente qui possède le paramètre.

Affichage des données vidéo

Plusieurs options sont disponibles pour visualiser des données vidéo lors de l'utilisation d'une intégration personnalisée :

  • Activités des spectateurs (événements) - Fourni dans le cadre du module Campagne, cette option peut être utilisée pour afficher les données vidéo à l'intérieur du module Campagne. Pour plus d'informations, voir Affichage des événements de lectuer dans le module de campagne.
  • Exporter des données - Fourni dans le cadre du module Campagne, cette option peut être utilisée pour exporter la vue vidéo et les données de prospect. Pour plus d'informations, voir Exportation des données de campagne à partir du module de campagne.
  • API Audience - L'API Audience peut être utilisée pour récupérer par programmation des vues vidéo et des données de piste. Pour plus d'informations, consultez Généralités: API d'audience.
