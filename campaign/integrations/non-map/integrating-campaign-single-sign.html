Le module Campaign offre la possibilité de suivre l'activité de visualisation lorsque le spectateur s'est connecté à un système à l'aide de l'authentification unique (SSO). Dans ce scénario, l'affichage des données n'est pas envoyé à une plate-forme d'automatisation marketing. Avec une configuration SSO de Campaign, les attributs SAML sont transmis à Campaign pour identifier l'utilisateur. Un cas d'utilisation courant pour cela serait les vidéos de formation internes. Les employés peuvent être tenus de se connecter pour regarder des vidéos, puis leur activité de visionnage peut être suivie puis visualisée dans le module Campagne. Pour activer l'utilisation de Campaign avec l'authentification unique, les étapes suivantes sont requises :

  1. Configurer le logiciel d'authentification unique
  2. Configurer votre application pour accéder aux attributs SSO
  3. Configurer la connexion SSO de la campagne
  4. Configurer l'intégration
  5. Créer un lectuer activé pour la campagne
  6. Afficher les données vidéo

Configuration du logiciel SSO

Les étapes d'installation et de configuration du logiciel SSO ne sont pas couvertes par cette rubrique. Le logiciel SSO doit être configuré de sorte que les utilisateurs doivent s'authentifier avant de pouvoir visualiser les vidéos Video Cloud. Le logiciel SSO doit être configuré pour exposer certains attributs utilisateur afin qu'ils puissent être accessibles à partir d'une application externe. Par exemple, cette configuration SSO Okta expose trois attributs FirstName, LastName et Email.

Dans ce sujet, le Email L'attribut sera utilisé pour identifier les téléspectateurs dans le module Campagne.

Note: Même si votre logiciel SSO peut être configuré pour exposer plusieurs attributs, Campaign n'utilise qu'un seul attribut pour suivre les spectateurs.

Configuration d'une application pour accéder aux attributs SSO

L'application ou le site Web qui sera utilisé pour visionner les vidéos Video Cloud doit être configuré de manière à ce que les attributs SSO soient mis à la disposition de Campaign à l'aide de JavaScript. Cette configuration varie en fonction de l'application ou du site Web utilisé. Dans l'exemple suivant, un profil de contrôle d'accès SSO a été créé dans Gallery. Dans le cadre de la configuration, l'attribut Email SSO est en cours d'exposition et sera disponible dans le site Gallery. C'est la valeur qui sera utilisée pour identifier les téléspectateurs du module Campagne.

Note: Pour plus d'informations sur la configuration des profils de contrôle d'accès dans Gallery, reportez-vous à la section Configuration des profils de contrôle d'accès.
Note: Même si Gallery peut être configuré pour exposer plusieurs attributs, Campaign n'utilise qu'un seul attribut pour suivre les spectateurs.

Configuration de la connexion

Procédez comme suit pour configurer la connexion entre Video Cloud et votre fournisseur SSO.

Établissement de la connexion entre Video Cloud et SSO

  1. Ouvrez le module Campagne et cliquez sur Connexions de données dans la navigation de gauche.
  2. Cliquez sur Add New Collection.
  3. Sélectionnez SSO comme intégration.
    Note: Les intégrations disponibles sont basées sur votre abonnement Video Cloud.
  4. (Facultatif) Entrez un nom de connexion.
  5. Cliquez sur Connect.

La connexion SSO est maintenant configurée et prête à l'emploi.

Configuration de l'intégration

Une fois la connexion SSO établie, vous pouvez configurer l'intégration.

  1. Clique le empty lien dans le Propriété SSO section. Entrez le nom de la valeur JavaScript à rechercher dans la page. Dans cet exemple, Gallery a été configuré pour exposer l'attribut Email SSO de sorte que c'est ce qui est utilisé ici.

    Dans cette rubrique, l'intégration de Campaign SSO sera utilisée pour suivre l'activité de visualisation. Cliquez sur Save.

    Note: Il est également possible de laisser la propriété SSO vide, puis d'utiliser l'une des options supplémentaires pour identifier les spectateurs.
  2. Par défaut, l' état de la connexion sera activé. L' état de la connexion détermine si les données d'engagement vidéo Video Cloud sont enregistrées.
    • Activé - Les données sont collectées
    • Désactivé - Aucune donnée n'est collectée (l'intégration est désactivée)
  3. Sélectionnez une politique de conservation des données. La politique de conservation des données détermine la durée pendant laquelle les données de visualisation et les données du formulaire de prospect (si vous utilisez un formulaire de prospect Campaign) seront conservées dans votre compte Video Cloud. Vous devez ajuster les paramètres de rétention en fonction de votre politique de confidentialité.

Pour supprimer l'intégration, cliquez sur Déconnecter. Le bas de la page répertorie tous les lectuers activés pour la campagne, les connexions auxquelles ils sont associés et les formulaires de prospects associés.

Le module Campaign prend en charge plusieurs connexions aux plateformes marketing. Cliquez sur Add New Collection pour configurer des intégrations supplémentaires.

Options supplémentaires pour identifier les spectateurs

Si vous ne spécifiez pas de valeur pour la propriété SSO lorsque l'intégration est configurée, vous êtes responsable de l'identification des visionneuses. Le module Campagne a besoin des informations suivantes :

Description Paramètre d'URL Paramètre d'intégration Montant
Quel est le GUID (nécessaire pour le suivi des événements de vue) bcssoguid data-bc-sso-guid string

Ce paramètre peut être transmis via un paramètre URL ou via le code d'intégration du lecteur. Pour que le suivi des visionneuses se produise, ce paramètre doit être passé en tant que paramètre URL sur la page qui affiche la vidéo.

Note: Pour que le lecteur/plugin lise le paramètre URL, le code d'intégration Advanced (in-page) doit être utilisé. Le code intégré Standard (iframe) ne fonctionnera pas car le plugin s'exécutera à l'intérieur d'un iframe et ne peut donc pas accéder à l'URL de la page parente qui possède le paramètre.

Il est également possible de transmettre le GUID au lecteur Campaign à l'aide d'une valeur JavaScript disponible dans la page. En utilisant la configuration du lecteur, il est possible de dire à Campaign quel est le nom de la variable JavaScript et Campaign tentera ensuite de lire cette variable en l'utilisant comme GUID personnalisé. Pour ce faire, éditez le lecteur Campaign et configurez les options de plug-in pour le plug-in Campaign.

Dans l'exemple ci-dessus, la configuration du lecteur indique à Campaign qu'il existe un objet JavaScript dans la page appelée samlAttributes et il a une propriété appelée nameID. Campaign tentera de le lire et d'utiliser sa valeur pour le GUID personnalisé.

Création d'un lecteur compatible avec la campagne

Les données d'engagement vidéo ne seront suivies que lorsque vous utilisez un lecteur vidéo compatible avec Campaign. Pour plus d'informations sur la création d'un Brightcove Player activé pour la campagne, consultez Création d'un lecteur compatible avec la campagne.

Affichage des données vidéo

Plusieurs options sont disponibles pour visualiser des données vidéo lors de l'utilisation d'une intégration SSO :

  • Activités des spectateurs (événements) - Fourni dans le cadre du module Campagne, cette option peut être utilisée pour afficher les données vidéo à l'intérieur du module Campagne. Pour plus d'informations, voir Affichage des événements de lectuer dans le module de campagne.
  • Profils de visionneuse - Fourni dans le cadre du module Campagne, cette option peut être utilisée pour afficher des données vidéo en fonction d'une valeur de profil utilisateur à l'intérieur du module Campagne. Pour plus d'informations, voir Affichage des informations de profil de la visionneuse
  • Exporter des données - Fourni dans le cadre du module Campagne, cette option peut être utilisée pour exporter la vue vidéo et les données de prospect. Pour plus d'informations, voir Exportation des données de campagne à partir du module de campagne.
  • API Audience - L'API Audience peut être utilisée pour récupérer par programmation des vues vidéo et des données de piste. Pour plus d'informations, consultez Généralités: API d'audience.
