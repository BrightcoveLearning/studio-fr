Le module Players permet de générer le code d'intégration du lecteur et l'URL du lecteur pour les lectuers en prévisualisation et publiés. Généralement, vous générez le code d'intégration d'un lecteur à l'aide du module Players lorsque vous souhaitez attribuer dynamiquement une vidéo au lecteur lors de l'exécution. Pour plus d'informations sur l'attribution dynamique de vidéos à un lecteur, consultez Attribution dynamique de vidéos à un lecteur.

Si vous êtes un client Video Cloud qui souhaite publier une playlist ou une seule vidéo sur un lecteur, utilisez la fonction de publication dans le module Media. Pour plus d'informations, consultez Publication de vidéos et de listes de lecture avec le module multimédia.

Comparaison lecteur publié/lecteur de prévisualisation

Le lecteur de prévisualisation permet de prévisualiser les modifications avant de les publier. Une fois les modifications vérifiées, le lecteur peut être publié et les modifications seront propagées à tous les lecteurs publiés. Le lecteur de prévisualisation affiche un logo de Brightcove en filigrane pour rappeler qu'il ne doit pas être mis en production.

Le lecteur publié a été optimisé et est le lecteur qui s'affichera sur vos pages Web. Ce lecteur contient toutes les modifications apportées aux paramètres du lecteur et publiées en cliquant sur le bouton Publier bouton.

Ouvrez le module lectuers et sélectionnez un lectuer. Cliquez sur Code intégré et URL et cliquez Aperçu du lecteur ou lectuer publié.

Types de code d'intégration

Il existe deux types de code intégré qui peuvent être générés :

  • Standard (iframe) - Recommandé dans la plupart des cas. Place le lectuer dans un <iframe> marque. Si vous avez spécifié les dimensions du lecteur, l'iframe aura les mêmes dimensions. Si aucune dimension n'a été spécifiée, l'iframe peut être placée dans une balise <div> et la nature réactive du lecteur fait qu'il se redimensionne pour occuper la totalité de l'espace de <div>.
  • Avancé (IN page): permet au lecteur d'exister directement dans la page HTML, et non dans un iframe. Cela permet aux développeurs d'accéder facilement au lecteur et aux propriétés et événements associés. En utilisant le code intégré avancé, vous perdez la facilité d'utilisation du lecteur comme une seule unité contenue dans les applications de médias sociaux et le partage. Cette option n'est disponible que pour le lecteur publié.

Générer du code d'intégration pour un lecteur de prévisualisation

Pour générer le code d'intégration d'un lecteur d'aperçu :

  1. Ouvrez le module Players.
  2. Cliquez sur la ligne du lectuer pour lequel vous souhaitez générer le code. Ne cliquez pas sur le nom du lecteur et ouvrez la page des propriétés.
  3. Cliquez sur Embed Code & URL puis sur Preview Player.
  4. Sélectionnez le code d'intégration approprié.

Générer du code d'intégration pour un lecteur publié

Pour générer le code d'intégration d'un lecteur publié :

  1. Ouvrez le module Players.
  2. Cliquez sur la ligne du lectuer pour lequel vous souhaitez générer le code.
  3. Cliquez sur Embed Code & URL puis sur Published Player.
  4. Sélectionnez le code d'intégration approprié.

Le code d'intégration du lecteur publié peut également être copié en cliquant sur le nom d'un lectuer, puis en cliquant sur Publier et intégrer.

Ajout d'un ID d'application au code d'intégration

Auparavant, il était courant pour les éditeurs de créer des lecteurs portant des noms différents afin de les utiliser sur diverses parties d'un site ou sur des sites différents. Le fait de disposer de plusieurs noms de lecteur facilitait la détermination des lecteurs les plus visionnés. En ajoutant un paramètre ID d'application au code d'intégration, le même lecteur peut être utilisé sur un ou plusieurs sites, et l'ID d'application peut être utilisé pour identifier l'endroit où le lecteur a été publié. L'attribution d'un ID d'application à un lecteur peut contribuer à réduire le nombre de lecteurs à créer et à gérer. Pour plus d'informations sur l'ajout d'un ID d'application au code d'intégration, voir Ajout d'un ID d'application au code intégré du lecteur.
