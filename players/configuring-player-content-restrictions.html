Le lectuer Restriction de contenu Les propriétés peuvent être utilisées pour restreindre un lectuer à des pays particuliers ou bloquer un lectuer de certains pays. Des restrictions de domaine peuvent être ajoutées à un lecteur pour limiter les domaines dans lesquels le lecteur peut être utilisé.

Notes :

  • Les Restriction géographique les propriétés ne sont disponibles que pour les comptes Entreprise et Professionnel. Restriction de domaine est disponible pour tous les comptes.
  • L'adresse IP peut également être utilisée pour limiter la lecture. Restriction de la lecture vidéo à l'aide de l'adresse IP.
  • IE 10 et 11, en mode de compatibilité, affichera les erreurs de restriction de domaine lors de l'affichage du contenu restreint au domaine.

Pour afficher et modifier le Restriction de contenu propriétés d'un lectuer, ouvrez le module lectuers, cliquez sur le nom d'un lectuer et cliquez sur Restriction de contenu dans le menu de navigation de gauche.

Note: Toute modification apportée à un lectuer peut prendre jusqu'à 5 minutes pour apparaître dans les lectuers publiés.

Configuration des restrictions de domaine

Les restrictions de domaines peuvent uniquement être mises en place sur les lecteurs pour limiter les domaines dans lesquels les lecteurs peuvent être utilisés. Si le code de publication du lecteur est copié et utilisé sur un autre site, les restrictions de domaines empêcheront le lecteur de charger des vidéos. Les éditeurs doivent utiliser le code intégré dans la page pour réellement faire fonctionner la lecture sur les domaines qui ont été mis en liste blanche. Les éditeurs ne peuvent pas utiliser le code d'intégration iframe sur un domaine de liste blanche (ou tout autre domaine) et s'attendent à ce que la lecture fonctionne correctement.

Notes :

  • Bien que les restrictions de domaine puissent aider à empêcher la distribution non autorisée de votre contenu, les clients qui recherchent une sécurité maximale du contenu devraient plutôt se tourner vers les solutions DRM. Voir Gestion des droits numériques (DRM) dans Video Cloud pour plus d'informations.
  • Pour les spectateurs utilisant Internet Explorer, la fonctionnalité de restriction de domaine nécessite la version 10 ou plus récente. Les utilisateurs d'une version d'Internet Explorer antérieure à la version 10 ne pourront pas regarder la vidéo.
  • Les restrictions de domaine sont prises en charge lors de l'utilisation d'une Galerie In-Page Experience et du code d'intégration recommandé (JavaScript).
  • Les lecteurs à accès restreint ne sont pas compatibles avec les intégrations AMP. Cela est dû au fait que l'intégration AMP crée un iframe qui, à son tour, n'est pas pris en charge par les lecteurs restreints par domaine.
  • Lorsque vous utilisez la fonctionnalité de restriction de domaine, les restrictions de domaine doivent être configurées pour tous les lectuers de votre compte. Si tous les lecteurs ne sont pas configurés avec des restrictions de domaine, quelqu'un peut prendre la clé de stratégie d'un lecteur non protégé et l'utiliser pour lire n'importe quelle vidéo.
  • Bien qu'il n'y ait pas de limite quant au nombre de domaines pouvant être ajoutés dans Studio, la longueur de l'URL de la demande d'API est limitée. Cette URL contient la liste des domaines délimitée par des virgules. Par conséquent, il y a une limite sur le nombre de domaines qui peuvent être spécifiés, mais le nombre dépendra de la durée des noms de domaine.

Pour configurer une restriction de domaine pour un lecteur, procédez comme suit :

  1. Entrez une ou plusieurs URL de domaine complètes (telles que http://www.brightcove.com ou https://www.brightcove.com) pour restreindre la disponibilité de votre lecteur à ces domaines particuliers. Veillez à inclure soit http ://soit https ://. Cliquez sur +.

    Notes :

    • Des URL complètes doivent être fournies. Les caractères spéciaux ne sont pas pris en charge. Les URL ne doivent pas se terminer par un /.
    • Seuls les domaines, et non les chemins spécifiques, sont pris en charge. Par exemple, vous pouvez ajouter https://www.brightcove.com, mais pas https://www.brightcove.com/documentation
  2. Cliquez sur Save pour enregistrer les modifications apportées.

Configuration du géofiltrage

Le géofiltrage peut être utilisé pour contrôler quels pays ont le droit (ou non) de lire la vidéo.

Notes :

  • Le filtrage géo est une fonctionnalité disponible pour les comptes Enterprise et Professional moyennant des frais supplémentaires. Contactez Brightcove si vous désirez activer le géofiltrage dans votre compte.
  • Le filtrage géographique peut également être appliqué à des vidéos individuelles. Video Cloud appliquera des politiques de filtrage géo des lecteurs et des vidéos. Si un pays est exclu par l'un ou l'autre, la vidéo n'y sera pas disponible.

Pour définir les propriétés de géofiltrage d'un lecteur :

  1. Sélectionnez Autoriser l'affichage dans les pays suivants ou Empêcher l'affichage à partir des pays suivants.
  2. Saisissez le nom des pays pertinents. Au fur et à mesure que vous tapez, la liste des pays sera filtrée.
  3. Cliquez sur un pays, puis appuyez sur Entrée pour l'ajouter à la liste.
  4. Cliquez sur Save.
