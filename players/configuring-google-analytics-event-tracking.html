Le module Players permet d'intégrer facilement les lecteurs Brightcove à Google Analytics. Cela permet d'envoyer des événements de lectuer et des données de dimension personnalisées à un compte Google Analytics à des fins de reporting. Notez que lorsque l'intégration est configurée, seules les analyses des événements vidéo seront transmises à Google Analytics.

Note: Chaque système d'analyse recueille les données et calcule les mesures différemment. Les données d'analyse Video Cloud ne seront pas identiques à celles qui apparaissent dans Google Analytics.
Note: Le plugin Google Analytics n'est pas pris en charge lors de l'utilisation des kits SDK Brightcove Player.
Note: Le module Players prend en charge le modèle de données Universal Analytics de Google. La nouvelle mise à jour de Google Analytics 4 n'est pas prise en charge. Voir le Versions de Google Analytics section ci-dessous pour plus d'informations.

Configuration de Google Analytics

Pour configurer l'intégration de Google Analytics, procédez comme suit :

  1. Ouvrez le module Players.
  2. Cliquez sur le nom du lecteur que vous souhaitez configurer pour ouvrir les propriétés du lecteur.
  3. Cliquez sur Analyse tierce> Google Analytics dans le menu de navigation de gauche.
  4. Cochez Activer Google Analytics.
  5. Cliquez sur Se connecter au compte Google.
  6. Sélectionnez le compte Google avec lequel vous souhaitez vous connecter et entrez vos informations d'identification si vous y êtes invité.
  7. Cliquez sur Autoriser pour permettre à Brightcove d'accéder à votre compte Google.
  8. Utilisez la liste déroulante Sélectionner un compte pour sélectionner le compte Google vers lequel envoyer les données de suivi.
  9. Sélectionnez les événements pour le suivi. Reportez-vous à la section Événements et dimensions personnalisées ci-dessous pour obtenir une description des événements.
  10. Cliquez sur Save. Cela permettra de configurer le lecteur avec le plugin Google Analytics et de créer les dimensions personnalisées nécessaires dans votre compte Google Analytics.
    Note: Vous devez avoir le Éditer autorisation dans votre compte Google Analytics afin de créer des dimensions.

Assurez-vous de publier les modifications sur le lecteur.

Note: Utilisez le code intégré avancé (dans la page) lors de la publication de votre lecteur. Le code d'intégration standard (iframe) ne fonctionnera pas car le plugin s'exécutera à l'intérieur d'un iframe et ne sera donc pas en mesure d'interagir avec le JavaScript nécessaire.

Événements & dimensions personnalisées

Les événements et les données de dimension personnalisées peuvent être transmis à Google Analytics.

Les événements suivants peuvent être passés.

Évènements basiques

  • Impression vidéo - Envoyée lorsqu'une vidéo a été chargée dans un lecteur et qu'elle est prête à être diffusée
  • Demande de lecture - Envoyée lorsque la lecture est lancée soit par l'utilisateur clique expressément sur le bouton de lecture, soit automatiquement lorsque la plate-forme déclenche la lecture dans un scénario de lecture automatique
  • Engagement vidéo: un utilisateur a regardé une plage de secondes de la chronologie d'une vidéo. Cet événement est un battement de cœur pour le suivi de l'engagement vidéo et sera probablement envoyé plusieurs fois pendant la lecture, en fonction de l'interaction de l'utilisateur et de la durée de la vidéo.

Évènements publicitaires

  • Début publicitaire - Envoyé lorsque le contrôle est confié à un agent publicitaire par la plateforme de lecture
  • Ad End - Envoyé lorsque le contrôle est remis par l'agent publicitaire à la plateforme de lecture

Evènements avancés

  • Charge du lecteur - Une session de lecteur a été lancée par un utilisateur final
  • Erreur - Envoyé lorsque des erreurs fatales qui perturbent l'expérience de lecture sont rencontrées

Dimensions personnalisées

Les dimensions personnalisées vous permettent de combiner des données Google Analytics avec des données autres que Analytics. Les dimensions personnalisées collectent des données que Google Analytics ne suit pas automatiquement. Les dimensions personnalisées suivantes sont disponibles.

Note: Pour plus d'informations sur les dimensions personnalisées, consultez l'aide de Google Analytics.
  • Secondes de la vidéo visionnée - Décompte des secondes regardées (disponible uniquement si le Engagement vidéo événement a été sélectionné)
  • Plage: la plage de secondes affichées (disponible uniquement si l'événement Video Engagement a été sélectionné)
  • Durée de la vidéo - La durée de la vidéo en secondes (disponible uniquement si le Engagement vidéo événement a été sélectionné)
  • ID du lectuer et nom du lectuer - ID et nom du lectuer
  • Nom du compte - ID de compte Brightcove
  • Session - Identificateur de session ; la valeur de session doit être constante pour le cycle de vie d'une expérience et envoyée pour tous les événements de cette expérience
  • Version lectuer - version Brightcove Player

Versions de Google Analytics

Le module Players prend en charge le modèle de données Universal Analytics de Google. Actuellement, la nouvelle mise à jour de Google Analytics 4 n'est pas prise en charge. Brightcove recommande d'utiliser Universal Analytics lors de l'intégration avec le module Lecteurs et les lecteurs Brightcove.

Voici les étapes de haut niveau pour créer une propriété Universal Analytics dans Google Analytics.

  • Créer une propriété
  • Montrer les options avancées
  • Basculer Créer une propriété Universal Analytics on (l'interrupteur à bascule est à droite)
  • Entrez une URL de site Web
  • S'assurer Créer une propriété Universal Analytics uniquement est sélectionné
  • Remplissez le reste du formulaire. Vous recevrez un nouvel identifiant de propriété commençant par UA -

Notez que Google offre également la possibilité de créer une sélection hybride avec à la fois une propriété Google Analytics 4 et Universal Analytics. Brightcove n'a pas entièrement testé ce scénario.
