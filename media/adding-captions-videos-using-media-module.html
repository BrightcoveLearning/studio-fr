Les propriétés des pistes de texte peuvent être configurées pour fournir les sous-titres d'une vidéo. Les sous-titres offrent des informations supplémentaires ou explicatives aux personnes qui souhaitent y accéder. Les sous-titres permettent aux sourds et aux malentendants de profiter de la partie audio des vidéos. Pour afficher les sous-titres d'une vidéo dans le lecteur Brightcove, vous devrez effectuer les tâches suivantes :

  • Créer des fichiers de légendes
  • Configurer les propriétés de la piste de texte
  • Légendes de style

Création de fichiers de sous-titres

Pour ajouter des sous-titres à une vidéo, vous devrez d'abord créer un fichier de sous-titres. Le lecteur Brightcove prend en charge les formats Web Video Text Tracks Format (WebVTT), SRT, DFXP et SCC. Le format de fichier WebVTT (illustré ci-dessous) est en texte brut et contient des données temporelles suivies du texte de légende ou de sous-titre souhaité.

Notes

  • Les sous-titres SRT, SCC et DFXP seront automatiquement convertis en WebVTT pour les comptes Dynamic Delivery.
  • Le texte des légendes doit être codé UTF-8 pour s'afficher correctement pour les langues sur deux octets comme le japonais et le coréen.
  • Étant donné que les légendes sont affichées en HTML, tout texte à l'intérieur < > les parenthèses seront interprétées comme une balise et ne seront pas affichées.
WEBVTT 00:00:00.000 --> 00:00:07.080 aligner:ligne médiane:90% 00:00:07.080 --> 00:00:10.280 align:middle line:84% Bonjour, je suis Bob Bailey, spécialiste de l'apprentissage chez Brightcove. 00:00:10.280 --> 00:00:12.780 align:middle line:84% Dans cette vidéo, vous découvrirez comment Video Cloud Studio fonctionne 00:00:12.780 --> 00:00:14.940 align:middle line:84% organisé et obtenez un bref aperçu de ce que 00:00:14.940 --> 00:00:17.490 align:middle line:90% vous pouvez accomplir en l'utilisant.
Notes :
  • Les noms des fichiers de sous-titres ne doivent inclure que des lettres (majuscules ou minuscules), des nombres, des tirets (-) et des traits de soulignement (_).
  • Nous avons vu des cas où les sous-titres créés par un tiers commencent les sous-titres avec l'horodatage :

    01:00:00 .000

    Cela entraînera les légendes une heure dans la vidéo. Pour vous assurer que les sous-titres sont correctement synchronisés avec la vidéo, assurez-vous que les horodatages commencent par 00:00:00 .000 et vérifiez tous les horodatages pour le décalage de 01 heure.

Pour plus d'informations sur le format WebVTT, cliquez sur ce lien. Vous devriez avoir un fichier de sous-titres distinct pour chaque langue que vous devez prendre en charge. Bien qu'il existe beaucoup d'outils tiers pour vous aider à créer des fichiers de sous-titres, Brightcove dispose de plusieurs partenaires qui proposent des services de conversion, de sous-titrage et de traduction. Visitez la section Partenaires de notre site internet pour obtenir la liste complète des partenaires Brightcove.

Configuration des propriétés des pistes de texte

Une fois les fichiers de sous-titres créés, ils peuvent être chargés et hébergés par Brightcove ou résider sur une URL accessible via internet.

Pour afficher et modifier les propriétés des pistes de texte d'une vidéo, ouvrez le module Médias, cliquez sur le lien de titre de la vidéo et localisez la section Text Tracks. Pour configurer les propriétés des pistes de texte, procédez comme suit :

  1. Cliquez sur Modifier ou double-cliquez dans la section.
  2. Sélectionnez si le fichier de suivi de texte sera téléchargé sur Video Cloud ou hébergé sur une URL externe.
    • Téléchargement de fichiers: cliquez sur Parcourir et sélectionnez le fichier de piste de texte dans le système de fichiers. Le fichier sera téléchargé dans Video Cloud.
    • Chargement d'URL: entrez l'URL publique où réside le fichier de piste de texte, puis cliquez sur Charger. Le fichier sera téléchargé dans Video Cloud.
    • URL distante: entrez l'URL publique où réside le fichier de piste texte. L'URL doit compter moins de 250 caractères. Les sous-titres seront chargés depuis l'URL distante.
      Note: Si vous hébergez des fichiers de sous-titres sur votre propre serveur, assurez-vous que le fichier se trouve à une URL accessible par le lecteur Brightcove. Le partage de ressources Cross-Origin Resource Sharing (CORS) doit également être activé sur le serveur hébergeant le fichier de sous-titres. Vous pouvez en savoir plus sur CORS ici.
      Note: Les noms de fichiers de suivi texte avec des parenthèses, des points d'interrogation, des guillemets, des espaces et d'autres caractères spéciaux échoueront silencieusement lors du téléchargement. Brightcove recommande d'utiliser des noms de fichiers sans caractères spéciaux.
      Note: La piste de texte des vidéos partagées sur le compte ne peut pas être modifiée dans le module Media.
    Note: Les Par défaut (affichage automatique) L'option n'a actuellement aucun effet sur les titres Dynamic Delivery.
  3. Le champ Language doit contenir le code de langue à deux lettres qui spécifie la langue des données du texte de la piste.
  4. Saisissez le libellé (Label) approprié du fichier de sous-titres. Le libellé est affiché dans le lecteur et permet au visiteur de sélectionner une langue. Si aucun libellé n'est mentionné, le code de langue s'affiche.
  5. Sélectionnez le type de piste de texte. Voici quelques options :
    • Légendes - Similaire aux sous-titres, mais peuvent également inclure des effets sonores et d'autres informations audio
    • Sous-titres - La transcription ou la traduction du dialogue
    • Descriptions - Destiné à être un fichier texte distinct qui décrit la vidéo via un lecteur d'écran
    • Chapitres - Destinés à aider l'utilisateur à naviguer dans la vidéo
    • Métadonnées - Informations et contenus concernant la vidéo qui ne sont pas destinés à être affichés par défaut par le spectateur (bien que vous souhaitiez peut-être le faire en utilisant JavaScript)
  6. Saisissez le type MIME du fichier de piste de texte. Sélection défaut affichera les légendes lors de la lecture de la vidéo (Brightcove Players 6.x et plus récent).
    Notez que si vous téléchargez des sous-titres SRT ou DFXP, vous ne pourrez pas modifier le type MIME.
  7. Cochez Default (Auto Display) pour faire de la piste la valeur par défaut. La plupart des navigateurs essaient d'afficher en premier la piste de texte par défaut, mais les paramètres de langue et d'accessibilité peuvent remplacer le paramètre par défaut.
    Note: Le paramètre par défaut ne fonctionne pas avec IE11.
  8. Cliquez sur Ajouter une piste de texte pour enregistrer les informations de piste de texte.
  9. Cliquez sur Add Text Track pour enregistrer les informations de piste de texte ou sur Save pour enregistrer les pistes de texte.
    Note: Les modifications apportées aux propriétés de la piste de texte peuvent prendre jusqu'à 15 minutes avant d'apparaître dans les lecteurs publiés.

Si des pistes de texte ont été désignées comme pistes par défaut, elles s'afficheront en gras. Pour supprimer une piste de texte, cliquez sur l'icône de suppression () à côté de la piste de texte.

Notez que le default la valeur n'est pas par piste, mais réglée sur true pour toutes les pistes de la même langue (srclang)

Lorsqu'un lecteur avec des sous-titres associés est publié, un bouton CC s'affiche sur le chrome du lecteur. Cliquez sur le bouton CC pour sélectionner une langue de sous-titrage et afficher les sous-titres.

Personnalisation du style des sous-titres

L'apparence par défaut des sous-titres est sujette à modification ; c'est pourquoi vous pouvez préférer leur donner un style. Pour plus d'informations sur le style des légendes, voir Styliser les légendes.

Contraintes

  • Les sous-titres sont pris en charge sur les types de dispositifs et de systèmes d'exploitation suivants.
    • iOS 10+
    • Android 7+
    • Internet Explorer 11+
    • Les dernières versions des navigateurs de bureau Chrome, Safari et Firefox
  • Il n'y a pas de limite technique à la longueur d'une légende, mais pour que la légende soit visible, le lectuer doit être assez grand pour afficher autant de texte
  • Étant donné que Safari utilise les fonctionnalités natives pour le style des légendes, l'option de menu Paramètres des légendes n'est pas disponible dans le menu CC des navigateurs Safari (iOS et macOS).
  • Si vous ingérez un fichier WebVTT non valide dans Studio, le fichier sera "traité", mais ne sera pas ajouté à la vidéo. Solution: vous pouvez valider vos fichiers WebVTT à l'aide de ce validateur en ligne.
