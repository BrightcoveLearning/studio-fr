Introduction

Il est possible d'ajouter des sous-titres aux vidéos offrant des informations supplémentaires ou explicatives aux personnes qui souhaitent y accéder. Les sous-titres permettent aux sourds et aux malentendants de profiter de la partie audio des vidéos.

Ce texte peut être utilisé à des fins d'accessibilité, de traduction de la piste audio ou d'autres fins, mais la technologie sous-jacente utilisée est la même.

Emplacement du fichier de sous-titres

Vous pouvez choisir où héberger le fichier de sous-titres WebVTT :

  1. Sur une URL distante (ressource distante) - Dans ce cas, le fichier WebVTT est hébergé sur votre serveur et vous ajoutez l'URL du fichier aux métadonnées vidéo.
    Note: Si vous hébergez des fichiers de sous-titres sur votre propre serveur, assurez-vous que le fichier se trouve à une URL accessible par le lecteur Brightcove. Le partage de ressources Cross-Origin Resource Sharing (CORS) doit également être activé sur le serveur hébergeant le fichier de sous-titres. Vous pouvez en savoir plus sur CORS ici.
  2. Chargé à l'aide de Video Cloud (ingéré) - Dans ce cas, le fichier WebVTT est téléchargé (ingéré) vers Video Cloud. Cela peut être fait à l'aide du module Media ou à l'aide des API.
Note: Si le fichier source vidéo que vous ingérez dans Video Cloud contient des sous-titres 608/708, ils seront ignorés lors du traitement de la vidéo ne pas présent dans les rendus vidéo. Pour avoir des sous-titres avec vos vidéos, vous devez les ajouter via le module Media, l'API d'ingestion dynamique ou l'API CMS (pour les sous-titres hébergés à distance)

Associer des légendes à une vidéo

Il existe plusieurs façons d'associer un fichier de sous-titres à une vidéo :

  1. Utilisation de Studio
  2. Modification du code de publication du lecteur
  3. Utilisation des API

Ajout de légendes dans Studio

Les légendes peuvent être associées à une vidéo à l'aide du module Media. Pour plus d'informations, consultez Ajout de légendes aux vidéos à l'aide du module multimédia.

Modification du code de publication du lecteur

Dans ce cas, un <track> La balise est ajoutée au code de publication du lecteur. Pour plus d'informations, consultez Ajout de légendes aux vidéos par programme.

Ajout de légendes via les API

La façon dont vous ajoutez des sous-titres via les API varie selon que le fichier de sous-titres sera ingéré ou utilisé comme ressource distante.

  • Légendes de ressources distantes - Si vous préférez héberger les sous-titres vous-même (ou si un fournisseur de sous-titres les héberge), vous pouvez associer les légendes à une vidéo à l'aide de l'API CMS. Pour plus d'informations, voir Utilisation de l'API CMS: Ajouter des légendes WebVTT.

  • Ajouter des sous-titres à Video Cloud - Si vous souhaitez que Brightcove héberge vos sous-titres, vous pouvez ingérer votre fichier WebVTT à l'aide de l'API Dynamic Inest. Pour plus d'informations, consultez Ingestion de fichiers WebVTT.

Note: Quelle que soit la méthode utilisée pour ajouter des fichiers de légendes à une vidéo, pas plus de 1000 pistes de texte peuvent être ajoutées.

Suppression des légendes

Les légendes peuvent être supprimées des vidéos à l'aide du module Media ou via l'API CMS. Dans Studio, ouvrez le module Media et modifiez les propriétés vidéo. Modifier le Pistes de texte propriété au besoin.

Les sous-titres peuvent également être supprimés à l'aide de la API CMS en faisant un Mettre à jour la vidéo demande et soumettre un corps de demande avec un vide text_tracks tableau (ou un tableau dont l'objet associé à cet ensemble de légendes a été supprimé).

Personnalisation du style des sous-titres

Le style peut être appliqué aux sous-titres WebVTT à l'aide de CSS. Voir Styliser les légendes pour plus de détails.
