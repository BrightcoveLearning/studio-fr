Lorsque vous utilisez les API Video Cloud, vos appels de méthode nécessitent des jetons d'accès, sauf si vous activez l'authentification de base dans les paramètres du service du lecteur. Pour plus d'informations sur l'activation de l'authentification de base, consultez la rubrique Gestion des paramètres du compte.

Note: Cette fonctionnalité est uniquement disponible pour les comptes Professional et Enterprise.

Création d'inscriptions client

La page API Authentification permet de créer des inscriptions de clients. Lors de la création d'inscriptions, vous pouvez sélectionner le service et les rôles qui seront activés avec l'inscription.

Pour créer une inscription pour votre compte :

  1. Cliquez sur le lien ADMIN dans l'en-tête Studio.
  2. Cliquez sur le lien API Authentification. La page Authentification API s'ouvrira et affiche vos enregistrements clients actuels.
    Note: Seules les inscriptions du client pour le compte auquel vous êtes connecté s'affichent.
  3. Cliquez sur Register New Application.
  4. Entrez un nom et une description pour l'enregistrement du client.
  5. Sélectionnez un ou plusieurs de vos comptes pour autorisation.
  6. Sélectionnez les API et les autorisations à activer pour cette inscription.
    Remarques:
  7. Pour l'intégration DFP/Google Ad Manager, les autorisations d'API nécessaires sont Lecture de vidéo CMS.
  8. La sélection d'un grand nombre de comptes et d'API peut entraîner la taille de l'en-tête HTTP devenir volumineuse et non valide. La taille maximale d'en-tête acceptée par notre serveur est de 8K, ce qui signifie que vous devez sélectionner 30 comptes ou moins pour appliquer les informations d'identification. Pour des raisons de sécurité, nous vous recommandons de limiter la portée des autorisations pour un ensemble d'informations d'identification au plus petit ensemble de comptes et d'opérations nécessaires.
  9. Cliquez sur Save.
  10. Un ID client et un secret client seront retournés.
    Note: Copiez le secret client et stockez-le dans un emplacement sécurisé car le secret client ne sera plus affiché.

Gestion des inscriptions client

La page API Authentification permet d'afficher et de gérer les inscriptions client. Cliquez sur le lien ADMIN dans l'en-tête Studio, puis cliquez sur Authentification API. Une liste de vos inscriptions actuelles s'affiche.

Pour afficher les détails de l'enregistrement, cliquez sur le nom. À partir de là, vous pouvez :

  • Modifier le nom
  • Copier l'ID client
  • Ajouter des comptes pour l'autorisation
  • Activer des API supplémentaires
  • Supprimer les informations d'identification

Cliquez sur Save pour enregistrer les modifications apportées.

Notez que si des informations d'identification sont créées pour plusieurs comptes, les utilisateurs ne verront ces informations d'identification que s'ils ont accès à tous ces comptes. Par exemple, l'utilisateur 1 crée des informations d'identification pour les comptes A et B. L'utilisateur 2 se connecte et n'a accès qu'au compte A. L'utilisateur 2 ne verra pas les informations d'identification car il n'a pas accès au compte B.

Suppression des enregistrements de clients

Pour supprimer un enregistrement client, cliquez sur l'icône de suppression () à côté de l'enregistrement ou modifiez l'enregistrement et cliquez sur Supprimer l'enregistrement. La suppression d'un enregistrement le désactive définitivement et ne peut pas être annulée.

Travailler avec DFP

Si plusieurs comptes Video Cloud sont actuellement configurés en tant que sources de contenu dans DFP, vous disposez de deux options :

  1. Vous pouvez créer un ID client et un secret client uniques pour chaque compte Video Cloud configuré en tant que source de contenu dans DFP. Par exemple, si vous disposez de 10 comptes Video Cloud, vous aurez 10 applications nommées DFP dans vos paramètres d'authentification API dans Video Cloud.
  2. Vous pouvez créer un ID client unique et un secret client unique pour tous les comptes Video Cloud configurés en tant que sources de contenu dans DFP. Par exemple, si vous disposez de 10 comptes Video Cloud, vous auriez 1 application nommée DFP dans vos paramètres d'authentification API dans Video Cloud.

Que vous ayez choisi l'option #1 ou #2, pour chaque source de contenu configurée dans DFP, vous devez fournir un ID de compte unique. Par conséquent, vous ne réduisez pas le nombre de sources de contenu actuellement configurées dans DFP ; vous remplacez simplement le jeton de lecture par l'ID client et le secret client, qui peuvent être identiques pour toutes les sources de contenu ou uniques pour chacune d'elles.

L'option #1 prend plus de temps à l'avance. Toutefois, si l'ID client et le secret client sont identiques pour toutes les sources de contenu (comme avec l'option #2), l'ajout d'une nouvelle source de contenu sera un fardeau car vous devrez générer un nouvel ID client et un secret client et mettre à jour toutes les sources de contenu dans DFP. Brightcove recommande donc l'option #1.
