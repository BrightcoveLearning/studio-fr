Introduction

L'API Social vous permet d'obtenir le statut et l'historique du partage de vidéos sur les plateformes de médias sociaux. Ce démarrage rapide vous permet de consulter les demandes disponibles et leurs réponses, à l'aide de votre propre compte ou d'un exemple de compte Brightcove Learning Services.

Remarques sur l'exemple d'application

  • Il s'agit d'un démarrage rapide, destiné à vous donner un modèle pour créer une application autour de l'API Social. Il vous permet de tester les requêtes de base, mais n'utilise aucun des paramètres que ces requêtes peuvent utiliser. Pour les voir, regardez la référence de l'API.
  • L'application fait également des requêtes au API CMS pour obtenir les noms des vidéos renvoyées par la demande initiale de statut de partage social vidéo. Il s'agit uniquement de remplir les sélecteurs vidéo pour les autres requêtes, car l'API Social renvoie uniquement les identifiants vidéo, pas leurs noms.
Cet exemple et le code associé sont fournis comme guide pour le développement de votre production. Le code n'est pas certifié pour une utilisation en production ni officiellement pris en charge.

Obtenir des informations

Pour utiliser l'API Social, vous aurez besoin d'informations d'identification appropriées.

Le moyen le plus simple de les obtenir est d'utiliser le Studio Authentification administrateur/API rubrique, cliquez sur Enregistrer une nouvelle demande , et sélectionnez les autorisations sociales pour vos informations d'identification :

  • Cliquez sur Modifier sur CodePen dans le coin supérieur droit pour incorporer ce CodePen dans votre propre compte.

Code proxy

Afin de construire votre propre version de l'exemple d'application sur cette page, vous devez créer et héberger votre propre proxy. (Les proxy utilisés par Brightcove Learning Services acceptent uniquement les demandes des domaines Brightcove.) Un exemple de proxy, très similaire à celui que nous utilisons, mais sans les contrôles qui bloquent les demandes provenant de domaines non-BrightCove, peut être trouvé dans ce dépôt GitHub. Vous trouverez également des instructions de base pour l'utiliser là-bas, ainsi qu'un guide plus élaboré pour créer des applications autour du proxy dans Utilisation des API REST.
