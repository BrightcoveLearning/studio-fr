Messages d'erreur généraux

État Prénom Message Cause
401 UNAUTHORIZED Permission denied. Jeton d'accès OAuth manquant ou non valide. Si vous avez fourni un jeton d'accès, il peut ne pas avoir pour la portée appropriée pour cette demande
403 FORBIDDEN   Vous n'avez pas l'autorisation d'accéder à la ressource. Dans la plupart des cas, vous aurez une erreur dans l'URL de la requête.
404 RESOURCE_NOT_FOUND Resource does not exist. Vous avez demandé une ressource qui n'existe pas: vérifiez les erreurs d'orthographe et de chemin d'accès.
422   (Message varies according to the problem; examples:
  • "Page size is not within the range(1-100) 200"
  • "The since parameter must be an ISO-8601 datetime string"
)		 Vérifiez les paramètres de requête pour vous assurer que les valeurs sont valides
500   an internal error occurred Problème dans le système Brightcove: réessayez plus tard.

Erreurs de partage social

Les erreurs de partage social ne sont pas renvoyées en tant que réponse d'erreur distincte à la requête API, mais plutôt dans un errors tableau dans la réponse normale.

Si l'opération de partage a échoué, l'API "result": "ERROR" signale et renvoie un objet d'erreur contenant :

  • error_code- Le code d'erreur renvoyé enregistré par le serveur, à partir de la plateforme sociale, le cas échéant.
  • error_message- Le message d'erreur enregistré par le serveur ou renvoyé par la plateforme sociale, le cas échéant.
  • platform_error- Représentation JSON de l'objet erreur renvoyé par la plateforme sociale, le cas échéant.

Voici un exemple

"errors": [
  {
    "error_code": "UPLOAD_FAILURE",
    "message": "Upload error: Received Facebook error response of type OAuthException: (#100) sponsor_id requires publishing profile/page to be verified or branding partner. (code 100, subcode null)"
  }
]
